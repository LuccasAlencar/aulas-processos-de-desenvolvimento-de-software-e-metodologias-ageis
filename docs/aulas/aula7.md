## 🎯 Introdução ao Tema
Até meados dos anos 90, o desenvolvimento de software era como construir um prédio: você planejava tudo, comprava os materiais e só via o resultado pronto depois de meses ou anos. Se o cliente mudasse de ideia no meio do caminho, o prejuízo era gigante. Este era o **Modelo Cascata**.

Em 2001, em uma estação de esqui em Utah, 17 especialistas criaram o **Manifesto Ágil**, mudando para sempre como o software é feito no mundo moderno.

## 📚 Objetivos de Aprendizado
- Entender o contexto histórico da crise do software.
- Dominar os 4 pilares do Manifesto Ágil.
- Perceber a mudança de mentalidade (Mindset) do tradicional para o ágil.

---

## 🏗️ O Problema do Modelo Tradicional (Cascata)
No modelo tradicional, as etapas são rígidas:
`Requisitos -> Design -> Implementação -> Testes -> Manutenção`

**O risco:** Você só descobre que o software está errado na fase de testes, quando já gastou 80% do orçamento.

---

## 💎 Os 4 Valores do Manifesto Ágil
O manifesto não diz que o que está à direita é inútil, mas que o que está à **esquerda** é muito mais importante para o sucesso:

1. **Indivíduos e interações** MAIS QUE processos e ferramentas.
   *(Ter pessoas conversando resolve mais problemas do que ter o software de gestão mais caro do mundo).*
2. **Software em funcionamento** MAIS QUE documentação abrangente.
   *(É melhor entregar uma tela que funciona do que um manual de 200 páginas sobre uma tela que não existe).*
3. **Colaboração com o cliente** MAIS QUE negociação de contratos.
   *(O cliente deve ser um parceiro que constrói junto, não alguém que você só vê na hora de cobrar).*
4. **Responder a mudanças** MAIS QUE seguir um plano.
   *(Se o mercado mudou ou surgiu uma tecnologia nova, mude o plano!)*

---

## 💬 Debate: O Mundo Mudou
O Manifesto Ágil foi criado por desenvolvedores, mas hoje ele é usado no Marketing, no RH e até na Educação. 
**Pergunta para a turma:** Por que seguir um plano rigoroso de 1 ano pode ser perigoso para uma empresa como o Spotify ou a Netflix?

## 🛠️ Exercício de Fixação: Tradicional vs. Ágil
Classifique as situações abaixo como **Tradicional (T)** ou **Ágil (A)**:
1. ( ) O time se recusa a mudar uma cor de botão porque "não estava no documento assinado mês passado".
2. ( ) O time entrega uma versão básica do app em 15 dias para o cliente já ir testando.
3. ( ) A equipe passa 3 meses apenas escrevendo manuais antes de programar.
4. ( ) O desenvolvedor levanta da cadeira para tirar uma dúvida rápida com o colega em vez de enviar um chamado formal.

---

## ✅ Resumo da Aula
O Manifesto Ágil não é uma metodologia (como Scrum), mas uma **filosofia**. Ele prega que o software deve ser flexível, focado em pessoas e entregue de forma incremental.
