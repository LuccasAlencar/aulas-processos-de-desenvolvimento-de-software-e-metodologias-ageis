    Aula 3 - Tipos de requisitos

    Antes de começar

    Pensou em quantas vezes você faz login em aplicativos hoje? Cada vez que interage com um app, está lidando com requisitos funcionais e não funcionais. Mas o que exatamente são esses requisitos?

    O que você vai aprender nesta aula

    • Diferenças entre requisitos funcionais e não funcionais.
    • Desafios no levantamento de requisitos.
    • Como gerenciar mudanças nos requisitos durante o desenvolvimento.
    • A importância da prototipagem na criação de softwares.

    Tipos de Requisitos

    Olá pessoal! Já pararam pra pensar quantas vezes vocês fazem login em aplicativos hoje? Cada vez que interagem com um app, estão lidando com requisitos funcionais e não funcionais. Mas o que exatamente são esses requisitos?

    Requisitos Funcionais vs. Não Funcionais

    Os requisitos funcionais são as coisas que o software precisa fazer para ser útil. Por exemplo, quando você digita uma senha para entrar em seu Instagram ou Facebook, isso é um requisito funcional: o sistema precisa permitir a autenticação dos usuários.

    Por outro lado, os requisitos não funcionais dizem respeito à qualidade e ao desempenho do software, como velocidade de resposta e segurança. Se você nunca recebeu uma mensagem de erro suspeita ou um aviso sobre segurança enquanto usava esses apps, isso é graças aos cuidados que os desenvolvedores tomaram para garantir que o sistema seja seguro contra ataques cibernéticos.

    🤔 Para refletir: Como você acha que a falta de requisitos não funcionais pode afetar sua experiência no uso do Instagram ou Facebook?

    Atividade Prática

    Agora, pense em um app ou jogo que você usa regularmente. Liste pelo menos três requisitos funcionais e dois não funcionais desse software.

    E aí, como vêem o papel dos requisitos na criação de softwares? Isso nos leva à nossa próxima discussão sobre especificação de requisitos.

    Desafios no Levantamento de Requisitos

    Quando você usa um app que não funciona bem, já parou pra pensar em quem desenhou aquele aplicativo? Ou por que ele parece estar faltando algumas coisas importantes?

    Identificando os Desafios

    Um dos grandes desafios é entender o que as pessoas realmente precisam. Seja você desenvolvendo um novo jogo ou melhorando uma rede social, às vezes pode ser difícil saber exatamente o que os usuários querem.

    🤔 Para refletir: Como se sente quando lida com algo completamente novo e desconhecido?

    Gerenciando Expectativas

    Imagine que você está criando um aplicativo de receitas e precisa entender os gostos culinários dos usuários. É fácil perder tempo pensando em detalhes irrelevantes ou não fazer perguntas corretas.

    🤔 Para refletir: Como distinguiria as necessidades verdadeiras dos desejos pessoais durante o levantamento de requisitos?

    Atividade Prática

    Vamos praticar! Escolha um app que você usa regularmente. Faça uma lista rápida das coisas que você gosta e não gosta no aplicativo. Agora, pense em como descobriria essas informações se estivesse fazendo o levantamento de requisitos.

    Próximo Passo

    Agora que entendemos os desafios iniciais, como podemos usar essa informação para melhorar a comunicação com nossos usuários finais?

    Gestão de Mudanças

    Quando você está usando um aplicativo legal e percebe que precisa de uma nova função que ainda não existe, como essa solicitação é gerenciada? Isso faz parte do desafio diário dos desenvolvedores de software.

    Gerenciar mudanças nos requisitos

    Imagine que você está jogando um jogo incrível. Depois de algumas semanas, o criador do jogo decide adicionar uma nova missão secreta baseada no feedback dos usuários. Isso pode parecer legal para você como jogador, mas para os desenvolvedores por trás do jogo, essa alteração não é tão simples.

    🤔 Para refletir: Como acha que isso afeta o cronograma de desenvolvimento e o orçamento?

    Atividade: Responda

    Imagine que um cliente pediu para adicionar uma função "notificações personalizadas" em um aplicativo que já está sendo testado. Liste três maneiras como essa solicitação pode impactar a equipe:

    1. Impacto no tempo de desenvolvimento.
    2. Possível ajuste nos recursos necessários.
    3. Necessidade de revisão do design atual.

    Permanecendo na rota certa

    Gerenciar essas mudanças sem perder o foco original é um desafio constante, mas necessário para manter os projetos de software saudáveis e bem-sucedidos.

    🤔 Para refletir: Como acha que os desenvolvedores podem equilibrar novas solicitações com as metas originais?

    Prototipagem

    Você já reparou quantas vezes por dia você usa apps diferentes? Redes sociais, jogos, e-mails, tudo é desenvolvido passando por um processo chamado prototipagem. Esse é o momento em que os criadores testam suas ideias antes de lançar algo final.

    A prototipagem é uma técnica valiosa que permite aos desenvolvedores testar ideias antes de implementá-las. Isso ajuda a identificar problemas e melhorias antes do lançamento oficial.

    Criar um protótipo inicial para um site pode ajudar a identificar problemas de usabilidade e design antes do lançamento final. O que você acha? Por que não testa essa ideia criando um protótipo simples para uma página que já conhece? Use papel, blocos de notas ou ferramentas digitais.

    🤔 Para refletir: Como ter um protótipo antes do lançamento ajuda a melhorar o produto final?

    Escreva três frases sobre os tipos de requisitos e como eles afetam o desenvolvimento de software. Use suas próprios palavras.

    O que fica desta aula

    - Requisitos funcionais: especificações do que um sistema deve fazer.
- Requisitos não funcionais: características relacionadas à qualidade e desempenho do sistema.
- Desafios no levantamento de requisitos: entender as necessidades dos usuários finais.
